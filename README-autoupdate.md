# อาสาน้อย — ระบบอัปเดตข้อมูลอัตโนมัติ

แอปจะ **ตรวจข้อมูลใหม่ทุกครั้งที่เปิดเว็บ** โดยโหลดไฟล์ `data.json` (โดเมนเดียวกัน ไม่ติด CORS)
- ถ้าโหลดได้ → ใช้ข้อมูลสด
- ถ้าโหลดไม่ได้ (ออฟไลน์/ยังไม่มีไฟล์) → ใช้ข้อมูลสำรองที่ฝังใน `index.html`
- รวมข้อมูลแบบ **union-by-id**: ของใหม่เพิ่มเข้ามา, ของเดิมอัปเดตได้, **ของเดิมไม่มีวันหาย**

## ไฟล์ในชุดนี้
| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | ตัวเว็บแอป (โหลด data.json ทุกครั้งที่เปิด) |
| `data.json` | ข้อมูลสดที่แอปโหลด (สร้างโดย scraper) |
| `seed.json` | ฐานข้อมูลที่ดูแลด้วยมือ = จุดตั้งต้นของ scraper |
| `scraper.py` | ดึงประกาศสด (เริ่มจากศิริราช) แล้วเขียน `data.json` |
| `.github/workflows/update-data.yml` | ตั้งเวลารัน scraper อัตโนมัติวันละ 2 ครั้ง |

## วิธีติดตั้งบน GitHub Pages (repo: ringzero-rot.github.io)
1. วางไฟล์ `index.html`, `data.json`, `seed.json`, `scraper.py` ไว้ที่ราก repo
2. สร้างโฟลเดอร์ `.github/workflows/` แล้วนำ `update-data.yml` ไปวางไว้ในนั้น
3. ที่ **Settings → Actions → General → Workflow permissions** เลือก **Read and write permissions**
4. เข้าแท็บ **Actions** แล้วกด **Run workflow** เพื่อทดสอบครั้งแรก (หรือรอรอบตามเวลา)
5. เปิดเว็บ → จะเห็นบรรทัด “🟢 ตรวจข้อมูลใหม่ทุกครั้งที่เปิดเว็บ · อัปเดตล่าสุด …”

## ปรับความถี่
แก้ `cron` ใน `update-data.yml` (เวลาเป็น UTC, ไทย = UTC+7)
- วันละ 2 ครั้ง (ค่าเริ่มต้น): `0 0 * * *` และ `0 11 * * *`
- ทุก 6 ชม.: `0 */6 * * *`
- ทุกชั่วโมง: `0 * * * *`

> หมายเหตุ: การ "เช็คทุกครั้งที่เปิดเว็บ" คือแอปโหลด `data.json` ล่าสุดทุกครั้ง
> ส่วนความสดของตัว `data.json` ขึ้นกับรอบที่ scraper รัน (เบราว์เซอร์ดึงข้อมูลจากเว็บโรงพยาบาลสด ๆ เองไม่ได้ เพราะ CORS + บางเว็บบล็อกบอต จึงต้องมี scraper ฝั่ง GitHub Actions)

## ขยาย scraper
`scraper.py` ทำ **ศิริราช** (news.asp, ถอดรหัส TIS-620/cp874) เป็นตัวอย่างจริงแล้ว
เพิ่มแหล่งอื่นได้โดยเขียนฟังก์ชัน `enrich_xxx(items)` เพิ่ม เช่น ราชวิถี, CampHub ฯลฯ
(บางเว็บบล็อก user-agent บอต อาจต้องตั้ง header หรือใช้วิธีเฉพาะ)

## ทะเบียนแหล่งข้อมูล & ความครอบคลุม
`scraper.py` มีตัวแปร `SOURCES` = รายชื่อแหล่งจริง 15 แห่ง:

**ดึงอัตโนมัติ (auto=True) — 3 แห่ง**
- ศิริราช (news.asp) — ดึงตำแหน่งจริงทีละตำแหน่ง + วันปิดรับ
- **DekPort** (dekport.com/volunteer) — เว็บรวมจิตอาสานักเรียนสายสุขภาพ อัปเดตรายวัน → auto-discovery
- **VolunteerSpirit** (volunteerspirit.org) — เครือข่ายจิตอาสา มีลิสต์กิจกรรม → auto-discovery

DekPort/VolunteerSpirit ทำหน้าที่ "ค้นทั้งเน็ต" มาให้แล้ว scraper จะไล่ดึงลิงก์ที่มีคีย์เวิร์ดสายสุขภาพ (โรงพยาบาล/แพทย์/พยาบาล/ผู้ป่วย ฯลฯ) มาเพิ่มใน data.json อัตโนมัติ **โดยติดธง `unverified`** — แอปจะโชว์ป้าย "🔎 จากแหล่งรวม · ตรวจก่อนสมัคร" เพื่อความปลอดภัย (กันข้อมูล/วันที่ผิด)

**ต้องอัปเดตด้วยมือ (auto=False) — 12 แห่ง** (ประกาศผ่าน Facebook/Google Form ที่บอตอ่านไม่ได้): รามาธิบดี, ราชวิถี, วชิรพยาบาล, สภากาชาด, กระจกเงา, พุทธิกา, ค่ายจุฬาฯ, Siriraj Open House, CampHub, dekuni, AdmissionPremium, Eduzones

### ⚠️ ตัวดึง aggregator เป็น "เบต้า" — ต้องตรวจรอบแรก
ตัวดึง DekPort/VolunteerSpirit เขียนแบบ best-effort (parse โครงสร้างเว็บที่อาจเปลี่ยนได้) **หลังรัน Action ครั้งแรก ให้เข้าไปดู log ในแท็บ Actions**:
- ถ้าเห็น `[auto] DekPort: เพิ่ม N รายการ` (N>0) = ทำงาน มีรายการใหม่เข้า data.json
- ถ้า N=0 หรือ error = เว็บนั้นเปลี่ยนโครงสร้าง/บล็อก — แจ้งมาได้ เดี๋ยวปรับ selector ให้

### เพิ่มโครงการที่ "ยืนยันแล้ว" เอง (แม่นสุด)
แก้ `seed.json` เพิ่ม object ตามรูปแบบเดิม (ไม่ต้องใส่ `unverified`) แล้ว commit — จะขึ้นเป็นการ์ดปกติ (ไม่มีป้ายเตือน) และข้อมูลเดิมไม่หาย (union-by-id)
